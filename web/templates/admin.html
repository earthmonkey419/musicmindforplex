⚙️ Back Office

Sync jobs and maintenance. Each job streams live output. Use direct IP for long-running jobs.

⚡ Full Sync

Runs the complete pipeline: scan Plex for new files → ingest into DB → sync Last.fm → tag new tracks.

🔍 Scan Plex Library

Tells Plex to scan for new or changed files on disk. Plex handles the scan in the background.

🔄 Sync Plex Library

Pulls all tracks from Plex into SQLite. Safe to re-run — updates existing records.

{% if lastfm_enabled %}

🎵 Sync Last.fm

Pulls new scrobbles since last sync. Updates play counts and loved tracks.

{% endif %}

🏷️ Tag New Tracks

AI-tags any untagged tracks via OpenAI gpt-4o-mini. Skips already-tagged tracks.

🔗 Refresh Context Playlists

Rebuilds Your Afternoon, Weekend Flow, and Often Together playlists in Plex.

🎼 Tag Instrumentals

Uses OpenAI to classify tracks as instrumental or vocal. Skips already-classified tracks. ~$0.50-1.00 one-time.

🎼 Enrich Artists (MusicBrainz)

Looks up artist gender, country, and MusicBrainz ID via the free MusicBrainz API. No cost, ~2 seconds per artist. Skips already-enriched artists.

🤖 Enrich Artists (AI Fallback)

Fills in era via OpenAI for all artists, and gender/country/era for any MusicBrainz couldn't match. Run after MusicBrainz enrichment.

📋 Refresh Gap Analysis

Re-runs gap analysis against current library. Removes acquired artists, adds new ones.

🔗 Fingerprint New Tracks

Computes acoustic fingerprints for any track that doesn't have one yet. Fast, local, no rate limit — also runs automatically as part of Full Sync.

♻️ Copy-Forward Analysis

Two tracks with identical fingerprints are the same recording — copies existing Synapse/VI results to any duplicate that's missing them, skipping redundant analysis. Run this again after a Synapse pass to catch new duplicates.

🔍 Duplicate Report

Read-only report — groups tracks by identical fingerprint to show real duplicates in your library. Never modifies or deletes anything.

🎯 Resolve Various Artists

Identifies real performers for "Various Artists" compilation tracks via acoustic fingerprinting + AcoustID lookup, instead of unreliable text matching. Rate-limited (~2s/track) — can take a while on a large backlog.
